    API de redes sociales: Referencia de error

    En este tema se proporciona una referencia para errores de Social API.

    Mensajes de error generales

    Estado Nombre Mensaje Motivo
    401 UNAUTHORIZED Permission denied. Falta el token de acceso OAuth o no es válido. Si proporcionó un token de acceso, es posible que no tenga para el alcance apropiado para esta solicitud
    403 FORBIDDEN   No tiene permiso para acceder al recurso. En la mayoría de los casos, tendrá un error en la URL de la solicitud.
    404 RESOURCE_NOT_FOUND Resource does not exist. Solicitó un recurso que no existe: comprueba los errores ortográficos y de ruta.
    422   (Message varies according to the problem; examples:
    • "Page size is not within the range(1-100) 200"
    • "The since parameter must be an ISO-8601 datetime string"
    )    		 Compruebe los parámetros de consulta para asegurarse de que los valores son válidos
    500   an internal error occurred Problema en el sistema Brightcove: inténtelo de nuevo más tarde.

    Errores de intercambio social

    Los errores de uso compartido en redes sociales no se devuelven como una respuesta de error independiente a la solicitud de API, sino más bien en una errors matriz en la respuesta normal.

    Si la operación de uso compartido falló, la API informará "result": "ERROR" y devolverá un objeto de error que contiene:

    • error_code - El código de error devuelto registrado por el servidor, desde la plataforma social, en su caso.
    • error_message - El mensaje de error registrado por el servidor, o devuelto desde la plataforma social, en su caso.
    • platform_error - Representación JSON del objeto de error devuelto desde la plataforma social, en su caso.

    Aquí hay un ejemplo

    "errors": [
  {
    "error_code": "UPLOAD_FAILURE",
    "message": "Upload error: Received Facebook error response of type OAuthException: (#100) sponsor_id requires publishing profile/page to be verified or branding partner. (code 100, subcode null)"
  }
]

    Consulte la documentación de la plataforma social específica para obtener más información sobre los errores que devuelven:

    • YouTube
    • Facebook
    • Twitter
